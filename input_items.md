# 入力項目定義（初期ドラフト）

> ユーザーがアプリに入力する「クレーム内容の要点」の項目案です。実際の業務で使っている項目に合わせて追加・削除・修正してください。

## 1. 入力項目の考え方

- ユーザーは箇条書き・簡潔なメモ程度の情報を入力するだけでよく、**文章の整形はAIに任せる**ことを前提とする。
- 入力項目は「必須」と「任意」に分け、最低限の必須項目だけでも報告書のたたき台が作れるようにする。

## 2. 入力項目一覧（案）

### 2.1 基本情報（必須）

| 項目名 | 説明 | 入力例 |
|---|---|---|
| 受付日 | クレームを受け付けた日付 | 2026-07-09 |
| 発生日 | 問題が発生した日付（分かる場合） | 2026-07-05 |
| お客様名／取引先名 | クレーム元の名称（個人名の場合は要配慮） | 株式会社〇〇 様 |
| 対応担当者 | 社内で対応した担当者名 | 山田 太郎 |
| 対象商品・サービス名 | クレーム対象となった商品/サービス | 〇〇製品（型番××） |

### 2.2 クレーム内容（必須）

| 項目名 | 説明 | 入力例 |
|---|---|---|
| クレームの概要 | 何が起きたかの要点（箇条書き可） | 商品に傷があった／説明と異なる対応をされた 等 |
| 発生経緯 | どのような経緯で問題が発生したか（分かる範囲） | 配送中に破損した可能性 等 |
| お客様の要望 | お客様が求めている対応 | 返金、交換、謝罪、再発防止策の提示 等 |

### 2.3 対応状況（必須）

| 項目名 | 説明 | 入力例 |
|---|---|---|
| 現在の対応状況 | 現時点で何を実施済みか | お詫びの連絡済み／交換品を手配中 等 |
| 今後の対応予定 | 今後実施予定の対応 | ◯日までに交換品を発送予定 等 |

### 2.4 社内向け補足情報（任意）

| 項目名 | 説明 | 入力例 |
|---|---|---|
| 原因分析（暫定） | 社内で把握している原因（推測含む） | 検品工程での見落としの可能性 等 |
| 再発防止策（暫定） | 現時点で考えられる再発防止策 | 検品項目の追加、担当者への注意喚起 等 |
| 影響範囲 | 同様の問題が他にも発生していないか等 | 同ロットの他出荷分は◯件、確認中 等 |
| 社内共有先 | 報告書を共有すべき部署・担当者 | 品質管理部、営業部長 等 |
| 重要度／緊急度 | 対応の優先度 | 高／中／低 |

### 2.5 出力設定（必須）

| 項目名 | 説明 | 入力例 |
|---|---|---|
| 出力する報告書の種類 | 社外用のみ／社内用のみ／両方 | 両方 |
| ファイル名（任意指定） | 出力するWordファイルの名前 | 未指定の場合は自動命名（[output_format.md](output_format.md) 参照） |
